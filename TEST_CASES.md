# AreaInfoDisplayOnPause 手動テストケース

実機での動作確認用のチェックリスト。自動テストは無いため、すべて手動でJump King上で確認する。

## 参考ファイル

- Modログ: `C:\Program Files (x86)\Steam\steamapps\common\Jump King\ModLoadLog.txt`
- クラッシュログ: `C:\Program Files (x86)\Steam\steamapps\common\Jump King\crashlog.log`
- 設定ファイル: Modのdllと同じフォルダの`F.AreaInfoDisplayOnPause.Settings.xml`
- 進捗データ: Modのdllと同じフォルダの`F.AreaInfoDisplayOnPause.AreaProgress.xml`

## 0. 事前準備

- [x] `dotnet build -c Release`でクリーンビルドが成功する
- [x] デプロイ先の`AreaInfoDisplayOnPause.dll`が、ローカルでビルドした`bin/Release/AreaInfoDisplayOnPause.dll`とハッシュ(md5など)が一致している
- [x] デプロイ先フォルダに`0Harmony.dll`も存在している

## 1. Mod読み込み確認

- [ ] ゲーム起動後、`ModLoadLog.txt`に`Successfully called 'Before Level Load' method with name 'BeforeLevelLoad' in 'F.AreaInfoDisplayOnPause'`が出力されている
- [ ] `ModLoadLog.txt`に`F.AreaInfoDisplayOnPause`関連のエラー・例外メッセージが出ていない
- [ ] `crashlog.log`のタイムスタンプが今回のプレイ時間より古い(新規のクラッシュが追加されていない)

## 2. 設定項目の表示・基本動作

- [x] メインメニューの設定一覧に「Show Total」「Attempt Counter」「Personal Best」「Progression Detail」の4項目が表示される（mod全体を無効化する項目は無い）
- [x] ポーズ中のメニュー（Mods → AreaInfoDisplayOnPause）からも同じ4項目が表示・変更できる
- [x] 設定ファイルが存在しない状態でゲームを起動すると、デフォルト値（Show Total=突破後に表示, Attempt Counter=ON, Personal Best=ON）で動作する
- [x] 各トグルを変更すると`F.AreaInfoDisplayOnPause.Settings.xml`に即座に反映される
- [x] 設定を変更した状態でゲームを完全に再起動し、変更内容が保持されている
- [ ] **Show Total**をNEVERにした状態でMods設定画面を開き、その場でAFTER CLEAR/ALWAYSに切り替えても、テキストが枠からはみ出さない・テキストの途中に不自然な隙間が空かない（メニューを開き直さなくても、その場で枠が正しく再計算される）

## 3. エリア表示の基本動作

- [x] エリア範囲内にいる時、`エリア名 現在ページ/総数`または`エリア名 現在ページ/x`（総数非表示時）の形式で表示される
- [x] エリア定義の範囲外（次のエリアへ向かう途中の「すき間」）にいる時、`On the way...`と表示される
- [ ] エリアの境界（前のエリアの`end`と次のエリアの`start`が同じ画面）をまたいで移動した時、表示が正しく切り替わる（前のエリアの最終ページが欠落しない・重複しない）
- [x] `start`と`unlock`が異なるエリア（例: False Kings Keep）で、`unlock`の画面にいる時にページ番号が`1`から始まる（`2`から始まらない）
- [x] **Show Total = 常に表示**で、未突破のエリアでも`n/m`形式で総数が出る
- [x] **Show Total = 表示しない**で、突破済みのエリアでも`n/x`のまま総数が出ない
- [x] **Show Total = 突破後に表示**で、未突破は`n/x`、突破後は`n/m`に切り替わる

## 4. 挑戦回数(Attempt Counter)

- [x] 各エリアに初めて入った時、`(#1)`と表示される
- [x] あるエリアの下のエリアに**落ちて**戻っても、下のエリアの挑戦回数は増えない
- [x] 一度落ちた後、再びそのエリアに**登って**戻ると、そのエリアの挑戦回数が`+1`される
- [x] 一番最初のエリアは、（登っていなくても）一番最初の画面に落ちた時点で挑戦回数が`+1`される
- [x] 脇道（独立した`Location`として定義されている場合）は、本筋のエリアから見れば「下のエリア」と同じ扱いになる。脇道に**落ちて**から本筋のエリアに**登って**戻ると、本筋のエリアの挑戦回数が`+1`される（脇道を経由していても、実際に登って戻ってきた以上は正しく加算される、Main Babeで確認）
- [ ] 一番最初のエリア以外でも、エリアの`unlock`の少し下（`start`〜`unlock-1`、「すき間(on the way)」として表示される範囲）に**落ちて**から、同じエリアの`unlock`に**登って**戻ると、そのエリアの挑戦回数が`+1`される（脇道や別の既知エリアを経由せず、純粋にすき間を挟んだ場合の確認）
- [x] **Attempt Counter**をOFFにすると`(#n)`の表示が消える（カウント自体は裏で継続している）

## 5. 突破済み判定とPage数表示

- [x] あるエリアから、定義順で**直後**のエリアに実際に到達した時だけ、元のエリアが「突破済み」になる
- [x] 脇道に入って戻ってきただけでは、誤って「突破済み」にならない（REDCROWN WOODSの脇道でチェック）
- [x] 足場に足を着けず空中を通過しただけ（着地していない）の場合、到達判定・挑戦回数・突破済み判定のいずれも更新されない
- [ ] エリアの移動がワープであっても、そのエリアの**最後の画面に一度以上到達していて**、かつワープ先が実際に（訪問順=`Order`で見て）次のエリアであれば、正しく突破済みになる（`start`の数値順では直後ではない場合でも判定される。BoI: Intense heat wind hell, doodlebugで確認）
- [ ] エリアの最後の画面に到達していない状態で脇道（独立したLocation）に初めて入った場合は、上記のワープ用判定によって誤って元のエリアが突破済みにならない

## 6. Personal Best (PB)

- [x] エリア1→エリア2と進むと、PBがエリア2側に更新される（エリア1のページ番号がエリア2より大きい数字でも、PBはエリア2になる。sorveillance breach 84->moonlight mansion 24でチェック）
- [x] あるエリアの4枚目まで到達して5枚目に入っていない場合、PBが`エリア名 4`のように表示される（5ではない）
- [x] 一度PBになった後、そのエリアより下に落ちても、PB表示は変わらない（下がらない）
- [x] まだどのエリアにも到達していない状態（レベル開始直後）では、PB行が表示されない
- [x] **Personal Best**をOFFにすると、PB行が表示されなくなる
- [ ] Progression DetailがONの間、Personal Bestトグルがグレーアウトして変更できない

## 7. Progression Detail

- [ ] ONにすると、通常のエリア情報表示の代わりに「pb: エリア名 ページ番号」の行が一番上に表示される
- [ ] その下に、到達済みの全エリアが**最後に訪れたものから順に**（最初のエリアが一番下になるように）列挙される
- [ ] 各エリアの行が`エリア名 (#挑戦回数) 0h 0m 0s`の形式（プレイ時間はそのエリアに初めて到達した瞬間の値）で表示される
- [ ] 同じエリアに再訪問しても、ラップタイム（初到達時のプレイ時間）の表示は更新されない
- [ ] まだどのエリアにも到達していない状態でONにすると、「No areas reached yet」と表示される
- [ ] ONの間、Attempt CounterとPersonal Bestのトグルがグレーアウトして変更できない
- [ ] トグルを押した瞬間に表示枠のサイズが正しく再計算される（テキストが枠からはみ出ない・枠が不自然に大きすぎない）

## 8. 表示枠の見た目

- [ ] 枠の左右の余白がテキストに対して概ね中央に来ている（右側だけ大きく余っていない）
- [ ] 枠の上下左右の余白が、本家の「Objective」表示と比べて極端に大きくない
- [ ] ポーズを開き直すごとに、その時点のテキスト長に合わせて枠のサイズが正しく再計算される（前回開いた時より長い/短いテキストでも欠けたり余りすぎたりしない）
- [ ] MODの設定画面で、テキストがはみ出ない。特に、AFTER CLEAR以外に設定された状態で起動して、そのあとAFTER CLEARにしてもはみ出ないかをチェック。
- [ ] PB行（1行目）が現在地表示（2行目）より長い場合でも、PB行がはみ出ない（カスタムレベル`3397533142`で確認）
- [ ] エリア名に他mod向けの`{color="..."}`等のマーカーが埋め込まれているカスタムレベル（`3397533142`の`Subspace Spire`）でPBが有効な状態でも、PB行・現在地行が正しく改行され、横にはみ出さない（MoreTextOptions等の他modが入っていても）

## 9. データ永続化

- [ ] プレイ中に進んだ後、**Save & Exit → Continue**で、挑戦回数・突破済み・PB・Progression Detailの内容がすべて保持されている
- [ ] プレイ中に進んだ後、**ゲームを完全に終了して再起動 → Continue**で、同様にすべて保持されている（再起動で消えない）
- [ ] **Restart**（セーブ削除を伴う方）を行うと、該当レベルの進捗データがリセットされる
- [ ] 別のレベル（別のセーブ/カスタムレベル）に切り替えても、お互いのデータが混ざらない（レベルごとに独立している）
- [ ] more savesでセーブした場合の動作について検証。


## 10. 安全性確認(本体ファイル非破壊の確認)

- [ ] `JumpKing.exe`のタイムスタンプ/サイズが変化していない
- [ ] Jump King本体のディレクトリ(`C:\Program Files (x86)\Steam\steamapps\common\Jump King\`)直下に、このMod由来の新規ファイル（dll/設定/進捗ファイル以外）が追加されていない
- [ ] Jump King本体のディレクトリ内の既存ファイルが変更されていない(ログ・セーブデータ等、ゲーム自身が書き出すもの以外)

## 11. 既知の限界の確認(回帰目的、修正は不要)

- [ ] ワープを使って通常ではあり得ない順序でエリアに到達した場合の挙動を確認し、明らかな不整合（クラッシュ・PBの逆行など）が無いことだけ確認する（完全な正確性は保証されない仕様）
- [x] **(既知の仕様)** ワープで2つ以上先のエリアに飛んだ場合、ワープ元のエリアは「突破済み」にならない（定義順＝**画面番号(`start`)の昇順**で本当に1つ後ろのLocationに着地した時だけ突破済みにする、という脇道誤認防止の仕組みがそのまま働くため。`Order`（初回訪問順）は無関係）。隣接エリアへのワープ（画面番号順で本当に1つ後ろに着地する場合）は正しく突破済みになる。これは脇道の誤判定を防ぐための仕組みの裏返しであり、修正不要

## 12. Babe（エンディング）画面の特別表示

- [ ] バニラ本編で、42画面目（New Babe）にちょうどいる時、通常のエリア表示の代わりに`Babe`と表示される
- [ ] 同様に99画面目（New Babe Plus）・153画面目（Owl）でも`Babe`と表示される（3つとも区別せず同じ`Babe`）
- [ ] Babe画面にいる間、PB行（`PB: Babe`）・Progression Detailの`pb:`行も同様に`Babe`になる
- [ ] Babe画面から1画面でも離れる（先に進む・下に落ちる）と、**現在地表示**は通常のエリア表示に戻るが、**PB行・`pb:`行は`Babe`のまま変わらない**（一度到達したBabeはそのプレイ中ずっとPBとして表示され続ける仕様）
- [ ] Babe画面にいる間も、裏側の挑戦回数・突破済み・通常のPBの記録（そのBabe画面が属する本来のエリアの分）は通常通り継続して更新されている（`Save&Exit→Continue`後、もしBabeに到達していなければ表示されるはずのPBで確認）
- [ ] Babeに到達した状態で`Save & Exit → Continue`、またはゲームを完全に再起動しても、PB行は`Babe`のまま保持されている
- [ ] **Restart**（セーブ削除を伴う方）を行うと、Babe到達状態もリセットされ、PB行が通常表示に戻る
- [ ] **Personal Best**をOFFにした状態でBabeに到達しても、PB行自体が表示されない（OFF設定が優先される。Progression Detailの`pb:`行はこのトグルと無関係に表示されることも合わせて確認）
- [ ] カスタムレベル（`level_settings.xml`に`ending_screen`/`ending_screen_second`/`ending_screen_third`があるもの）でも、該当画面で`Babe`が表示される
- [ ] Babe画面に到達すると、その画面が属する本来のエリア（例: `main babe`の`The Tower`4枚目がBabeの場合は`The Tower`自身）が「突破済み」になる（**Show Total = 突破後に表示**設定で、Babeから離れた後の`The Tower`の表示が`n/m`形式に切り替わっていることで確認）
